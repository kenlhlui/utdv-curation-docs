title: Testing the U of T Dataverse Curation Tool
description: Instructions for testing the U of T Dataverse Curation Tool
---

1. **Identify the datasets you are responsible for here**: [PLACEHOLDER_LINK_TO_XLSX] \- go to the tab with your name to view the datasets that have been assigned to you. Pick a dataset and note the DOI, and the project ID of it.

2.  **Use the tool to review the dataset**: Start the curation tool if you haven't already (Ken will provide instructions). Enter the tool in your web browser at `http://localhost:9005`. Click the 'New Project' box. Input the DOI and Project ID to the corresponding fields. Click 'Submit'.

3. **Complete the curation log**: Once the tool finishes the check, it will redirect you to another the checklist. See the [features documentation](./features.md) for the tool's feature. Use the page to review the dataset and complete the curation checklist. These are the inputs you should provide in the checklist:  
      1. **Status**: Select the appropriate status for each item (Pass, Follow-Up, TBD, Not Applicable)
      2. **Comments**: Fill in the comments with the checklist item.
      3. **Time Spent**: Log the time spent on each item in the checklist. Click the start `▶︎` button to start the timer when you begin working on an item, and click the stop `■` button when you finish. The time will be logged in the "Time Spent" column. <br>If you forget to start the timer, you can also manually input the time spent in the "Time Spent" column in the format of MM:SS (e.g. 15:30 for 15 minutes and 30 seconds).

Tip: While a file is selected in Windows File Explorer, you can press `F2` to rename it.
